## Описание реализованных типов и контейнеров

### Generic-класс `TypedCollection[T]`

Обобщённая коллекция, которая знает, какой тип объектов в ней хранится.

| Метод | Аннотация | Описание |
|-------|-----------|----------|
| `add(item: T) -> None` | Добавление элемента |
| `remove(item: T) -> None` | Удаление элемента |
| `remove_at(index: int) -> T` | Удаление по индексу |
| `get_all() -> List[T]` | Получить все элементы |
| `find(predicate: Callable[[T], bool]) -> Optional[T]` | Поиск по условию |
| `filter(predicate: Callable[[T], bool]) -> List[T]` | Фильтрация |
| `map(transform: Callable[[T], R]) -> List[R]` | Преобразование с изменением типа |
| `sort_by(key_func: Callable[[T], Any], reverse: bool) -> TypedCollection[T]` | Сортировка |

### TypeVar

| TypeVar | Использование |
|---------|---------------|
| `T` | Тип элементов в коллекции |
| `R` | Тип результата `map` (может отличаться от T) |
| `D` | Тип с ограничением `bound=Displayable` |
| `S` | Тип с ограничением `bound=Scorable` |

### Протоколы (оценка 5)

| Протокол | Требуемый метод | Какие классы подходят |
|----------|-----------------|----------------------|
| `Displayable` | `display() -> str` | `Player`, `Warrior`, `Mage`, `Archer` |
| `Scorable` | `get_score() -> float` | `Player`, `Warrior`, `Mage`, `Archer` |
| `HealthProtocol` | `health`, `is_alive` | `Player`, `Warrior`, `Mage`, `Archer` |

**Важно:** Классы не наследуют эти протоколы! Они подходят благодаря наличию нужных методов — это и есть **структурная типизация**.

## 4. Вывод

В ходе лабораторной работы были изучены и применены:

### Аннотации типов
Позволяют указать ожидаемые типы данных, что делает код понятнее и помогает IDE подсказывать методы. Ошибки обнаруживаются раньше.

### Generic и TypeVar
Generic-класс `TypedCollection[T]` умеет хранить объекты любого типа, но запоминает его. При добавлении элемента не того типа IDE покажет ошибку — это безопаснее обычного списка. TypeVar `R` в методе `map()` демонстрирует, что результат может быть другого типа (например, из `List[Player]` получаем `List[str]`).

### Protocol (структурная типизация)
Класс подходит под протокол, если у него есть нужные методы — наследование не требуется. В работе объекты `Player`, `Warrior`, `Mage`, `Archer` соответствуют протоколам `Displayable` и `Scorable` благодаря наличию методов `display()` и `get_score()`, без явного указания.

### Итог
Типизация делает код более надёжным, самодокументированным и удобным для поддержки.
