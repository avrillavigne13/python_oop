# ЛР-4 — Интерфейсы и абстрактные классы (ABC)

## Цель работы

Познакомиться с абстрактными базовыми классами (ABC), освоить понятие интерфейса (контракта поведения), научиться задавать обязательные методы для классов, закрепить полиморфизм через единый интерфейс, научиться проектировать архитектуру, а не просто классы.

---

## Предметная область

**Игровая логика / RPG**

Базовая сущность: **`Player`** (Игрок) → интерфейсы: `Printable`, `Comparable`, `Damageable`

---

## Описание интерфейсов

В рамках лабораторной работы созданы 3 абстрактных класса (интерфейса), которые задают контракты поведения для игроков.

### Интерфейс `Printable`

Требует реализовать метод получения строкового представления объекта.

| Метод | Возвращает | Описание |
|-------|------------|----------|
| `to_string()` | `str` | Строковое представление объекта |

### Интерфейс `Comparable`

Требует реализовать метод сравнения объектов.

| Метод | Возвращает | Описание |
|-------|------------|----------|
| `compare_to(other)` | `int` | `-1` если текущий меньше, `0` если равны, `1` если больше |

### Интерфейс `Damageable`

Требует реализовать методы получения урона и лечения.

| Метод | Возвращает | Описание |
|-------|------------|----------|
| `take_damage(amount)` | `str` | Получить урон |
| `heal(amount)` | `str` | Вылечиться |

---

## Реализация в классах

Классы `Player`, `Warrior`, `Mage`, `Archer` наследуются от базовых классов из ЛР-3 и реализуют все три интерфейса.

### Класс `Player` (базовый)

| Метод интерфейса | Реализация |
|-----------------|------------|
| `to_string()` | `[Тип] Имя | Ур.N | Статус` |
| `compare_to(other)` | Сравнение по силе (уровень * здоровье%) |
| `take_damage(amount)` | Уменьшение здоровья, проверка смерти |
| `heal(amount)` | Увеличение здоровья (не более максимума) |

### Класс `Warrior` (воин)

| Метод интерфейса | Особенности реализации |
|-----------------|------------------------|
| `to_string()` | Добавляет броню и оружие |
| `compare_to(other)` | Учитывает броню в расчёте силы |
| `take_damage(amount)` | Урон уменьшается на процент брони |

### Класс `Mage` (маг)

| Метод интерфейса | Особенности реализации |
|-----------------|------------------------|
| `to_string()` | Добавляет ману и заклинание |
| `compare_to(other)` | Учитывает ману в расчёте силы |
| `add_experience()` (переопределён) | Получает на 20% больше опыта |

### Класс `Archer` (лучник)

| Метод интерфейса | Особенности реализации |
|-----------------|------------------------|
| `to_string()` | Добавляет меткость и тип лука |
| `compare_to(other)` | Учитывает меткость в расчёте силы |
