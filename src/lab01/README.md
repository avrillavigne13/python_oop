## Описание класса

В рамках лабораторной работы реализован пользовательский класс `Player`, который описывает игрока в RPG-игре.

Каждый объект класса хранит информацию о конкретном игроке:

- имя;
- уровень;
- здоровье;
- опыт;
- состояние (жив/мертв);
- титул (вычисляется автоматически).

---

## Реализованные возможности

### Атрибуты экземпляра

| Атрибут | Тип | Описание |
|---------|-----|----------|
| `_name` | str | Имя игрока |
| `_level` | int | Уровень (1–10) |
| `_health` | float | Здоровье (0–100) |
| `_experience` | int | Опыт (зависит от уровня) |
| `_is_alive` | bool | Состояние (жив/мертв) |

### Атрибуты класса

| Атрибут | Значение | Описание |
|---------|----------|----------|
| `MAX_LEVEL` | 10 | Максимальный уровень |
| `MIN_LEVEL` | 1 | Минимальный уровень |
| `MAX_HEALTH` | 100.0 | Максимальное здоровье |
| `EXPERIENCE_PER_LEVEL` | 1000 | Опыта для повышения уровня |
| `TITLES` | `{(1,2):"Новичок", ...}` | Титулы для диапазонов уровней |

---

## Инкапсуляция

Инкапсуляция реализована с помощью закрытых атрибутов экземпляра с одним подчёркиванием:

- `_name`
- `_level`
- `_health`
- `_experience`
- `_is_alive`

Доступ к этим полям организован через свойства (`@property`), а изменение здоровья и опыта выполняется через сеттеры с валидацией.

---

## Валидация данных

При создании объекта выполняется проверка входных данных.  
Логика проверки вынесена в отдельный модуль **`validate.py`**:

| Функция | Назначение |
|---------|------------|
| `validate_string()` | Проверка имени (не пустое, длина 2–20 символов) |
| `validate_int()` | Проверка целых чисел в диапазоне |
| `validate_float()` | Проверка чисел с плавающей точкой в диапазоне |
| `validate_experience()` | Проверка опыта относительно текущего уровня |

Проверяется:

- корректность типа данных;
- непустая строка для имени;
- допустимый диапазон уровня (1–10);
- здоровье в пределах 0–100;
- опыт не превышает максимум для текущего уровня (`level * 1000`).

## Свойства (`@property`)

| Свойство | Геттер | Сеттер | Описание |
|----------|--------|--------|----------|
| `name` | ✅ | ❌ | Только чтение |
| `level` | ✅ | ❌ | Только чтение |
| `health` | ✅ | ✅ | С валидацией (0–100) |
| `experience` | ✅ | ✅ | С валидацией, вызывает level up |
| `is_alive` | ✅ | ❌ | Вычисляется из здоровья |
| `title` | ✅ | ❌ | Вычисляется из уровня |
| `health_percent` | ✅ | ❌ | Процент здоровья |

Для свойств `health` и `experience` реализованы сеттеры с валидацией:

- нельзя установить здоровье вне диапазона 0–100;
- нельзя установить опыт, превышающий максимум для текущего уровня;
- нельзя добавить опыт мёртвому игроку.

## Бизнес-методы

| Метод | Описание |
|-------|----------|
| `take_damage(damage)` | Наносит урон игроку. При здоровье ≤ 0 игрок умирает. |
| `heal(amount)` | Лечит игрока. Не работает для мёртвого игрока. |
| `add_experience(amount)` | Добавляет опыт. При достижении порога повышает уровень и восстанавливает здоровье. |
| `revive()` | Воскрешает мёртвого игрока с половинным здоровьем. |
| `_level_up()` | Внутренний метод повышения уровня. |

---

## Логическое состояние объекта

В классе реализовано логическое состояние игрока:

- игрок **жив**;
- игрок **мёртв**.

### Управление состоянием

| Метод | Эффект |
|-------|--------|
| `take_damage()` | При получении урона здоровье уменьшается. Если здоровье становится ≤ 0, состояние меняется на "мёртв". |
| `revive()` | Воскрешает игрока, изменяя состояние на "жив". |

### Поведение, зависящее от состояния

- нельзя лечить мёртвого игрока;
- нельзя добавить опыт мёртвому игроку;
- мёртвый игрок не может получать урон;
- при повышении уровня здоровье восстанавливается до максимума.

# Ответы на вопросы по лабораторной работе

## Вопрос 1. Что является сущностью?

Сущностью является игрок (Player) в RPG-игре. Это активный участник игрового мира, который обладает характеристиками (уровень, здоровье, опыт), может взаимодействовать с окружающей средой (получать урон, лечиться) и развиваться (повышать уровень). В реальном мире аналогом может служить персонаж в компьютерной игре, которым управляет игрок.

---

## Вопрос 2. Какие у него атрибуты?

Класс Player имеет следующие атрибуты экземпляра: имя игрока (_name), уровень от 1 до 10 (_level), текущее здоровье от 0 до 100 (_health), текущий опыт от 0 до level * 1000 (_experience) и состояние жив или мертв (_is_alive). Также есть атрибуты класса, общие для всех экземпляров: MAX_LEVEL = 10, MIN_LEVEL = 1, MAX_HEALTH = 100.0, EXPERIENCE_PER_LEVEL = 1000 и словарь TITLES, который сопоставляет диапазоны уровней с титулами (Новичок, Воин, Ветеран, Легенда).

---

## Вопрос 3. Какие инварианты?

Инварианты — это условия, которые всегда должны соблюдаться для любого корректного объекта класса. В классе Player определены следующие инварианты: уровень всегда находится в пределах от 1 до 10, здоровье всегда от 0 до 100, опыт не превышает максимум для текущего уровня (level * 1000), состояние alive всегда соответствует здоровью (игрок жив только если здоровье больше 0), и имя не может быть пустым или короче 2 символов. Эти инварианты гарантируются проверками в конструкторе при создании объекта, валидацией в сеттерах при изменении атрибутов и автоматическим обновлением _is_alive при изменении здоровья.

---

## Вопрос 4. Что значит “равенство”?

В классе Player равенство определяется по имени игрока. Два игрока считаются равными, если у них одинаковые имена, независимо от уровня, здоровья, опыта и других характеристик. Например, игрок с именем "Артас" 5 уровня и игрок с именем "Артас" 3 уровня будут считаться равными, потому что это один и тот же персонаж. А игроки с именами "Артас" и "Тралл" будут разными. Такая логика выбрана потому, что в игровой механике имя обычно является уникальным идентификатором персонажа.

---

## Вопрос 5. Есть ли состояние?

Да, класс Player имеет явное логическое состояние, которое влияет на поведение объекта. Есть два основных состояния: ЖИВ (когда здоровье больше 0) и МЁРТВ (когда здоровье меньше или равно 0). Состояние влияет на все действия игрока: живого игрока можно лечить, он может получать урон и опыт; мертвого игрока нельзя лечить, он не может получать урон или опыт. Состояние может изменяться постепенно через получение урона (при health ≤ 0 игрок умирает) или явно через метод revive (воскрешение). Также можно выделить подсостояния на основе уровня здоровья (критическое, среднее, отличное), но на поведение влияет только флаг _is_alive.
